# Stakeholder Review Checklist

## Purpose
Ensure stakeholders have appropriate visibility and input at key project milestones. This checklist helps maintain accountability and prevents surprises during project execution.

Related to: [Issue #4 - Process Improvements](https://github.com/NihadZeidan/skills-scale-institutional-knowledge-using-copilot-spaces/issues/4)

## Pre-Review Preparation

**Project Manager Responsibilities:**
- [ ] Identify which stakeholders need to review (consult stakeholder list from Project One-pager)
- [ ] Schedule review meeting or set review deadline for async feedback
- [ ] Prepare review materials (summary deck, demo, or document)
- [ ] Share materials at least 2 business days before the review meeting

**Materials to Include:**
- [ ] Project status summary (progress, blockers, risks)
- [ ] Deliverables or demo ready for review
- [ ] Open questions or decisions needed from stakeholders
- [ ] Next steps and timeline

## During Review

**Facilitation (PM or Agile Coach):**
- [ ] Set context and objectives for the review
- [ ] Present deliverables (demo led by Developer or Designer as appropriate)
- [ ] Capture stakeholder feedback and concerns
- [ ] Document decisions and action items
- [ ] Clarify next steps and follow-up timeline

**Key Stakeholder Inputs:**
- [ ] Business value validation (Product Manager confirms alignment)
- [ ] User experience feedback (UX/UI Designer notes improvements)
- [ ] Quality assessment (QA Lead confirms acceptance criteria met)
- [ ] Requirements verification (Business Analyst validates completeness)

## Post-Review Actions

**Within 24 hours:**
- [ ] PM shares review notes and decisions with team
- [ ] Action items assigned with owners and due dates
- [ ] Updated timeline or scope documented (if changes agreed)
- [ ] Follow-up questions sent to stakeholders

**Within 1 week:**
- [ ] All action items addressed or in progress with status updates
- [ ] Stakeholders informed of action item status
- [ ] Risk register updated with any new risks identified
- [ ] Project plan updated to reflect approved changes

## Review Frequency by Phase

| Project Phase | Review Type | Stakeholders | Frequency |
|--------------|-------------|--------------|-----------|
| Initiation | Approval to proceed | Sponsor, Product Lead | One-time |
| Planning | Plan review & approval | Sponsor, PM, PdM, key stakeholders | One-time |
| Execution | Sprint/milestone demos | PdM, stakeholders, users (optional) | Every sprint or milestone |
| Pre-Release | Go/no-go decision | Sponsor, PdM, QA Lead, PM | Before each release |
| Post-Release | Retrospective & results | Team, PdM, stakeholders | After release |
